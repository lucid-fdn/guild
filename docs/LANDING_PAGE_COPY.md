## FAQ

Question:

Is Guild another agent framework?

Answer:

No. Guild is an orchestrator-agnostic control and institution layer above existing agent frameworks and runtimes.

Question:

Why not just use shared memory?

Answer:

Shared memory alone does not solve ownership, review, replay, approvals, or governed learning. Guild treats artifacts and institutional process as first-class.

Question:

Can Guild work with my existing stack?

Answer:

That is the point. Guild is designed to work with existing orchestrators through adapters and neutral spec objects.

Question:

What makes Guild different from orchestration tools?

Answer:

Orchestration tools focus on execution flow. Guild focuses on institutions: handoffs, ownership, artifacts, replay, and promotion of proven learnings.
