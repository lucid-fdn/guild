# Naming Decision

## Decision

Keep the public repository name as `guild` for the alpha.

## Why

- The repo already exists at `lucid-fdn/guild`.
- The name is short, memorable, and aligned with agent teams, roles, craft, and accountable work.
- The current wedge is not generic "institution software"; it is an agent work contract.
- Renaming now would slow dogfood and launch preparation more than it would improve clarity.

## Positioning

Use this sentence everywhere:

```text
Every agent run starts with a mandate and ends with proof.
```

Use `Guild` as the project name, `AgentDesk` as the local workflow, and `Taskpack` as the portable mandate object.

## Revisit Criteria

Revisit the name only before a major public launch if:

- the name conflicts with another active developer tool
- package names are unavailable
- early users consistently misunderstand the product category
- a clearly stronger name appears with available repo, package, and domain paths
