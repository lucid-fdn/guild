# v0.1.0-alpha.4

Fourth public alpha focused on making the GitHub issue intake loop fully CLI-driven.

## Install

```bash
go install github.com/lucid-fdn/guild/cli/cmd/guild@v0.1.0-alpha.4
guild agentdesk init
guild mcp serve
```

## New

- `guild agentdesk issue create "Title" --repo owner/repo --scope "docs/**"`
- CLI-created issues are labeled `agent:ready` and include objective, allowed scope, acceptance criteria, and agent notes.
- README and quickstart no longer require manually creating an `agent:ready` issue in GitHub UI.

## Copy-Paste Demo

```bash
go install github.com/lucid-fdn/guild/cli/cmd/guild@v0.1.0-alpha.4

GITHUB_TOKEN="$(gh auth token)" \
guild agentdesk bootstrap github --repo lucid-fdn/<repo>

GITHUB_TOKEN="$(gh auth token)" \
guild agentdesk issue create "Fix docs typo" \
  --repo lucid-fdn/<repo> \
  --scope "docs/**" \
  --acceptance "Docs are updated and proof is attached."

GITHUB_TOKEN="$(gh auth token)" \
guild agentdesk next --source github --repo lucid-fdn/<repo>
```

Then claim the mandate, attach proof, verify, export replay, and open a PR.
